Test 7: Reset All Functionality

🔄 Reset Test Steps:
  1. Enable JavaScript blocking → Apply → Reload (should block scripts)
  2. Click "Reset All" button in Page Controls
  3. Reload this page
  4. All tests should show 🟢 WORKING status
If JavaScript remains blocked after Reset All + reload, the resetAllControls() method is not properly calling chrome.runtime.sendMessage({ type: "unblockJavaScript" })

📊 Expected Behavior Summary

When JavaScript Blocking is ENABLED: • External .js/.mjs/.jsx files are blocked (network level) • Inline scripts continue to work (already parsed in HTML) • Toast shows "JavaScript blocking enabled" ONCE only When JavaScript Blocking is DISABLED: • Blocking rules are removed from declarativeNetRequest • Toast shows "JavaScript unblocked" ONCE only • After reload, external scripts load normally When Reset All is clicked: • All blocking rules are removed (including JavaScript) • originalState.javascriptBlocked set to false • After reload, page behaves normally with no restrictions
